# Meta-Team 量化团队设计方案

## 1. 概述

本设计基于论文 "Evolve as a Team: Collaborative Self-Evolution for LLM-based Multi-Agent Systems" 中的 Meta-Team 框架，为 A 股量化分析场景构建自进化多智能体系统。

### 1.1 需求摘要

- **市场**: A 股
- **核心任务**: 选股分析、投资组合优化、风险评估、回测验证、因子研究
- **数据源**: tushare + akshare
- **团队规模**: 3-5 人小团队
- **协作模式**: 混合模式（集中式编排 + 去中心化协作）

## 2. 架构设计

### 2.1 三层自进化框架

```
┌─────────────────────────────────────────────────────────┐
│                    Meta-Team 量化架构                      │
├─────────────────────────────────────────────────────────┤
│  L3: Team-Level (共享脚手架)                               │
│  - 量化章程 (团队目标、分析框架、风控标准)                      │
│  - 数据源配置 (tushare + akshare)                         │
│  - 协调规则 (信息交换协议、冲突解决)                          │
├─────────────────────────────────────────────────────────┤
│  L2: Interaction-Level (交互协议)                          │
│  - 数据流: Data Agent → Factor Agent → Portfolio Agent    │
│  - 风控反馈: Risk Agent ↔ Portfolio Agent                 │
│  - 回测信号: Backtest Agent → Portfolio Agent             │
├─────────────────────────────────────────────────────────┤
│  L1: Agent-Level (专业职能)                                │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐   │
│  │ Data Agent  │  │ Portfolio   │  │ Risk & Backtest │   │
│  │ 数据获取     │  │ Agent       │  │ Agent           │   │
│  │ + 因子研究   │  │ 选股+组合   │  │ 风控+回测       │   │
│  └─────────────┘  └─────────────┘  └─────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 2.2 Agent 职责

| Agent | 核心能力 | 输入 | 输出 |
|-------|---------|------|------|
| **Orchestrator** | 任务分解、全局调度、结果综合 | 用户请求 | 任务分配、结果汇总 |
| **Data Agent** | 数据获取、因子计算 | tushare/akshare API | 行情数据、因子矩阵 |
| **Portfolio Agent** | 选股、组合优化 | 因子数据、风控信号 | 推荐组合、权重 |
| **Risk & Backtest Agent** | 风控评估、回测验证 | 组合方案、历史数据 | 风险报告、回测结果 |

### 2.3 协作流程

```
1. 用户发起量化分析请求
2. Orchestrator 分解任务 → Data Agent + Portfolio Agent + Risk Agent
3. Data Agent 获取A股数据 → 计算因子矩阵
4. Portfolio Agent 基于因子选股 → 输出候选组合
5. Risk & Backtest Agent 评估风险 → 回测验证
6. 如果回测不通过 → 反馈给 Portfolio Agent 重新优化
7. Orchestrator 综合结果 → 返回给用户
8. Team-Level 定期总结：更新量化章程、优化协调规则
```

## 3. Meta-Team 自进化机制

### 3.1 进化层级

| 进化层级 | 触发条件 | 更新内容 |
|---------|---------|---------|
| **L1 (智能体级)** | 个股/因子表现不佳 | 选股因子、权重策略 |
| **L2 (交互级)** | 协作效率低 | 信息交换协议、依赖关系 |
| **L3 (团队级)** | 整体业绩差 | 量化章程、风控标准 |

### 3.2 协作自我进化方案

- 每个智能体携带本地执行上下文进入进化阶段
- 智能体主动通信，交换处理后的本地发现
- 追踪输出如何影响下游执行

## 4. 量化框架配置

### 4.1 因子体系

| 因子类别 | 指标 |
|---------|------|
| 价值因子 | PE、PB、PS、PCF |
| 成长因子 | 营收增速、净利润增速、毛利率变化 |
| 动量因子 | 20日收益率、60日收益率、120日收益率 |
| 质量因子 | ROE、ROA、资产负债率、存货周转率 |

### 4.2 风控参数

- 单股持仓上限: ≤20%
- 组合最大回撤: ≤15%
- 行业集中度: 单行业 ≤30%
